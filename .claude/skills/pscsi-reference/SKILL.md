本スキルは X68000 Z の pSCSI（pseudo SCSI）実装に関する技術仕様を参照しながら、実装・デバッグ・調査を支援する。

## 1. pSCSIシステム全体構造

pSCSI は Human68k 環境のストレージアクセスをホスト OS 上で最適化するための抽象化レイヤーである。物理 SCSI バスの忠実な再現ではなく、エミュレーション特有の制約を活かした設計。

### 主要モジュール

| モジュール | 配置 | 役割 |
|---|---|---|
| pSCSIデバイスヘッド | ROM/IPL内、`$00ea0000`〜 | ブート時のデバイス認識エントリーポイント |
| pSCSI IOCS | trap #15 / 機能コード `$f5` | SCSIデバイス制御の高位API |
| pSCSIコントローラ | I/Oレジスタ経由 | SPC ステートマシンの擬似再現、バスフェーズ遷移管理 |
| pSCSIデバイスハンドラ | 最下層 | HDSイメージファイルの直接操作、LBAベースの読み書き |

### 設計上の重要原則：DMAバイパス

実機の「サイクルスチールDMA」を排し、**Immediate Memory Copy（即時メモリコピー）**でデータ転送を実現。転送待ち時間をゼロに近づけ、エミュレータのメイン処理をブロックしない。

## 2. I/Oレジスタマップ

ベースアドレス：**`$00ea0020`**（実機ボード CZ-6BS1 や内蔵SCSI との競合回避）

| アドレス | レジスタ名 | R/W | 内容 |
|---|---|---|---|
| `$00ea0020.b` | `S_RESET` | W | コントロールレジスタ（bit7=1でコマンド実行） |
| `$00ea0020.b` | `S_STATUS` | R | ステータスレジスタ（bit7=1でビジー状態） |
| `$00ea0021.b` | `S_PHASE` | R | SCSIフェーズ識別（Data/Command/Status等） |
| `$00ea0022.b` | `S_SELECT` | R/W | ターゲットID（TID）の設定 |
| `$00ea0023.b` | `S_CMDOUT` | W | SCSIコマンドレジスタ |
| `$00ea0024.l` | `S_LBA` | W | 論理ブロックアドレス（LBA） |
| `$00ea0027.b` | `S_LEVEL` | R | 割り込み・準備レベル（**実機互換性に極めて重要**） |
| `$00ea0028.l` | `S_TCOUNT` | W | データ転送長（バイト単位） |
| `$00ea002c.l` | `S_MADDR` | W | 転送先/元メモリアドレス（物理アドレス） |
| `$00ea0030.w` | `S_MSGIN` | R | メッセージイン・バイト |
| `$00ea0032.w` | `S_STSINT` | R | ステータス・バイト |

### ベースアドレス比較（混同注意）

| 種別 | ベースアドレス |
|---|---|
| **pSCSI** | `$00ea0020` |
| 内蔵SCSI（MB89352） | `$00e96020` |
| 外付けボード（CZ-6BS1） | `$00ea0000` |

## 3. pSCSI IOCSコール仕様

エントリポイント：**`trap #15`、機能番号 `$f5`**、`d1.w` でサブコマンド指定

| `d1.w` | コール名 | 機能 |
|---|---|---|
| `$20` | `_S_INQUIRY` | ベンダー/製品情報（"SHARP "等）の取得 |
| `$21` | `_S_READ` | 指定LBAからのセクタ読み出し |
| `$22` | `_S_WRITE` | 指定LBAへのセクタ書き込み |
| `$23` | `_S_FORMAT` | 物理フォーマット（**ダミー実行・正常終了を返す**） |
| `$24` | `_S_TESTUNIT` | ターゲットのレディ状態確認 |
| `$25` | `_S_READCAP` | 容量・ブロックサイズ（512B）の取得 |
| `$29` | `_S_MODESENSE` | デバイスパラメータの取得 |
| `$2b` | `_S_REZERUNIT` | ドライブの再校正（ゼロシーク） |
| `$2c` | `_S_REQUEST` | 詳細エラー情報（Request Sense）の取得 |

### ダミー実装の互換性戦略

`_S_FORMAT ($23)` はエミュレーション上でフォーマット不要。ただし上位アプリ/インストーラが成功前提で動作するため、**HDSイメージのファイルサイズを維持したまま正常終了を返す**ことで互換性を確保する。

## 4. HDSファイル構造

X68000 SCSI ハードディスク構造をセクタ単位で模倣したバイナリイメージ。セクタサイズは **512バイト固定**。

### 論理ブロック（LB）構成

| LB | 内容 |
|---|---|
| **LB0** | SCSI BOOT IPL |
| **LB1〜2** | SCSIデバイスパラメータ（セクタサイズ/シリンダ/ヘッド数等） |
| **LB4〜5** | パーティションテーブル |

### LB0（SCSI BOOT IPL）

```
オフセット $00: "X68SCSI1" (8 bytes) ← 識別マジックナンバー
オフセット $40: bra 命令 → IPLロード先ジャンプ
```

### LB4〜5（パーティションテーブル）

```
オフセット $00: "X68K" (8 bytes) ← 識別マジックナンバー
最大15エントリ。各エントリ構成：
  $00: フラグ（$00=Bootable自動起動、$02=In use使用可能）
  開始LBA
  総ブロック数（1024バイト単位で管理）
```

## 5. Human68k 起動シーケンス

ROM内の pSCSI デバイスヘッドが実行する認識手順：

1. **TEST UNIT READY** (TID 0〜順次スキャン) → 応答確認
2. **REQUEST SENSE** → リセット直後の Unit Attention クリア、エラー状態リトライ
3. **READ CAPACITY** → ブロックサイズが 512バイトであることを検証
4. **LB0 読み込み** → `$00` に `"X68SCSI1"` を確認、オフセット `$40` から IPL ロード
5. **パーティションテーブル解析** → LB4〜5 を読み込み、有効な "Human68k" 領域（フラグ `$02`）をカウント
6. **ドライブ登録** → 有効パーティションに A:, B: 等を割り当て

### 起動高速化：TIDタイムアウト最適化

`pscsi.ini` で未定義の TID へのアクセスに**即座に「デバイス不在（-1）」を返す**ことで、実機（数秒間のフィジカル待ち）を超える高速起動を実現。

## 6. 既知の制約と実装上の罠

### S_LEVEL レジスタの罠（最重要）

Human68k ディスクドライバ **v1.04** は `S_LEVEL` が **4未満**の場合に「ハードウェア不正」と判断して**無限ループ**に陥る。

```
pSCSI実装では S_LEVEL を常に 4 以上の適切な値で返すこと。
0等の不適切な値を返すと OS 起動時にフリーズする。
```

### 非DMA転送の即時性

- DMA完了待ちのウェイト処理やタイミング依存コードは実機と異なる挙動を示す
- 転送はメモリコピーで即時完了するため、DMA割り込みを前提とするコードは要修正

### pscsi.ini によるTIDマッピング

- HDS ファイルと SCSI ID（0〜15）の紐付けは `pscsi.ini` に依存
- Human68k の認識順（TID 0 から順スキャン）と設定の整合性が必要
- 不整合があると起動ドライブの優先順位が逆転する

## 実装チェックリスト

実装・デバッグ時に以下を確認する：

- [ ] `S_LEVEL` レジスタは常に `4` 以上の値を返しているか
- [ ] LB0 のマジックナンバー `"X68SCSI1"` は正確に 8 バイトか
- [ ] パーティションテーブル（LB4〜5）の `"X68K"` マジックナンバーは正確か
- [ ] `_S_FORMAT` はファイルサイズを変えずに正常終了を返しているか
- [ ] 未定義 TID へのアクセスに即座に `-1` を返しているか
- [ ] `pscsi.ini` の TID マッピングと Human68k の認識順序が一致しているか
- [ ] セクタサイズが 512バイト固定になっているか
- [ ] `S_READCAP` で 512バイトのブロックサイズを正しく返しているか
